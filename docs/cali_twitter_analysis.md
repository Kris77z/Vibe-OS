# 基础架构启示录：给 AI 搭建「永不失忆」的记忆系统

本文档是对 Cali Castle 系列推文的深度提炼，作为 Vibe-OS 个人外脑系统的核心设计准则和第一性原理参考意见。

## 核心痛点与问题定调
*   **LLM 的短期失忆症：** 上下文窗口无论多大（如 200K / 1M），本质仍是临时内存（RAM）。重置会话即丢失。
*   **全量向量检索（RAG）的反模式：** 纯向量检索存在极大的信息噪音。人类记忆不靠“全量录像查找”，而是靠“经验蒸馏、压缩与遗忘”。如果把所有对话生吃不嚼塞进向量数据库，最终会导致检索失效和幻觉。

---

## 一、三层“仿生”记忆架构 (The Memory Layers)
这套系统本质是模拟人类大脑“清醒-睡眠-复盘”的巩固机制。

### Layer 1: 每日短记忆沉淀 (Daily Context Sync)
*   **触发机制：** 每日夜间定时任务（Cron Job）。
*   **行为逻辑：** 类似于人类睡觉做梦。调用推理成本较低的 LLM（如 Claude Sonnet），或者利用专门的文本总结 Prompt，读取一天内所有无序的零散对话。
*   **输出结果：** 提炼出结构化的 Markdown 档案，提取出核心决策和 Action Items。
*   **关键目录指向：** 生成的结构化档案存放在类似于 `memory/YYYY-MM-DD.md` 的路径。重新对这部分信息进行高密度的 Index 索引。

### Layer 2: 每周长记忆固化 (Weekly Memory Compound)
*   **触发机制：** 每周末定时运行。
*   **行为逻辑：** 类似于深度反思与知识结晶。将 Layer 1 产生的大量日常切片再度浓缩，剔除过期时效性细节。
*   **输出结果：** 形成长期维度的全局视图文档（Project Overviews / Status Reports）。

### Layer 3: 瞬时工作记忆保护 (Hourly Micro-Sync)
*   **触发机制：** 小时或更高频率的持续快照。
*   **行为逻辑：** 自动存档机制，防止系统或 Agent 进程崩溃导致当天的对话全部阵亡（类似于打游戏的 Auto-Save）。

## 二、零阻力的人机交互与流转体验 (Zero-Friction Workflow)
如果说上面的架构是“大脑存储器”，这部分就是“脑机接口”。

### 1. 消除书写阻力的“脑洞卸载口” (Brain Dump)
*   **核心痛点：** 传统笔记应用（Notion / Apple Notes / Obsidian）拥有极高的操作门槛与决策成本（找应用、想标题、分目录），这会极大程度打断心流，导致“记完即死（Dead Notes）”。
*   **解决方案：** 打造一个无格式、无分类、无需思考的收集箱极其关键。所有的想法、吐槽、待办、或者是洗澡/走路时的灵感，直接往里扔。
*   **关键文件指向：** 所有输入（Append-only）全部追加指向 `memory/braindump.md`。

### 2. 人工智能辅助 GTD (Mission Digestion)
*   **去人工干预：** 免去传统的 David Allen GTD 方法中痛苦的“清空收集箱（Inbox）”步骤。这些脏活全部下放给 AI。
*   **提纯转换流：**
    *   **输入端 (Input):** `memory/braindump.md` (杂乱无章的废话集合)
    *   **处理端 (Process):** 定期通过 LLM 清洗、去重、推断意图。
    *   **输出端 (Output):** 将想法变成可执行的任务列表，更新至 `memory/mission_log.md`。

### 3. 主动状态唤醒 (Active Context Injection)
*   **机制原理：** 检索不仅仅是“我搜你答”，而是系统的被动理解+主动提示。
*   **关键展现：** 当在讨论特定话题（比如新项目立项）时，系统应当能（借助向量召回或历史提炼）主动提供用户在一周前的深夜随口提及的相关点子和疑虑，即使这些点子已被用户自己遗忘。

---

## 三、关键系统文件/指令映射 (Key System Files & Directives)

在构建 Vibe-OS 时，以下文件节点必须被严格落实和映射：

1.  **全局状态强制注入层：`workspace/MEMORY.md`**
    *   **作用：** 系统的“最高宪法”和小抄（Cheat Sheet）。
    *   **包含内容：** 用户的基础设定、当前最高优先级的关注点、投资/工作核心法则。
    *   **调用方式：** 在每次新 Session 启动或核心流转前，强制作为 System Prompt 的一部分注入进 Context。

2.  **倾倒收集终端：`memory/braindump.md`**
    *   **作用：** 所有碎片信息的时间戳归档黑洞。
    *   **特殊指令：** 必须在 Agent System Prompt 中告诉它一条核心法则：“当用户只是随口倾倒想法时（标志：没有明确疑问句），无需作答或给出通用建议，只负责安静接收，打上时间戳并追踪至 `braindump.md`”。

3.  **行为准则：`AGENTS.md` (或同等配置文件)**
    *   **作用：** 在类似 OpenClaw 这种多 Agent 框架下，定义每个 Agent 的职责边界和行为准则（比如：谁负责聊天，谁负责静默爬文档）。

4.  **每日/每周提炼结果堆场：`memory/YYYY-MM-DD.md` 及 `memory/mission_log.md`**
    *   **作用：** 已经过 LLM 脱水处理的高密度信息结晶块，用于未来的准确 RAG 查询和任务执行。
